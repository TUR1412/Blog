> [!IMPORTANT]
> **卷规三条（写给读者，也写给写卷的人）**
> 1. 正文只围绕“修仙纪事与人物志”：专注“轩天帝（轩少）”的行止与章法。
> 2. 不写编程教程，不写无关杂谈；宁可少，也不乱。
> 3. 不夸大、不粉饰：要像真事——能落地、可对照、可追溯。

<a id="vol-0"></a>

## 卷首 · 起势

这不是“吹出来”的修仙。更像一卷被反复翻过的旧经：热闹在外，分寸在里；锋芒在言下，章法在行止。

> [!TIP]
> **入卷最快**：点击「在线洞天」直入；以灵镜（<kbd>Ctrl</kbd>/<kbd>⌘</kbd> + <kbd>K</kbd> 或 <kbd>/</kbd>）召出索引，直达一章、一事、一处去处。

> [!NOTE]
> **一句话写法**：此卷以“对照”为骨，以“分寸”为刃；不靠声势，靠细则。

<a id="vol-1"></a>

## 卷一 · 地界（所见与可走）

| 地界 | 所见与可做 | 一句看点 |
| --- | --- | --- |
| 洞天 | 进站、总览、直达 | 第一眼先定“气口” |
| 纪事 | 按章读、收藏、续读 | 写得像真事的起承转合 |
| 洞府图 | 年表路标、筛选、定位 | 把岁月写成路 |
| 关系谱 | 人/誓词/旧物/关口/地点 | 一张“对照之网” |
| 批注馆 | 汇总批注、检索、直达 | 把线索摊开，慢慢看 |
| 札记 | 即写即存、经卷观、卷内检索 | 把分寸留在卷里 |
| 藏品 | 拖拽排序、记住次序 | 旧物自有来处 |

> [!NOTE]
> 此卷的“厉害”不靠夸张：靠的是把一件事写清，把一个人写稳。

<a id="vol-2"></a>

## 卷二 · 手感（交互要 Q，要顺）

- **灵镜检索**：<kbd>Ctrl</kbd>/<kbd>⌘</kbd> + <kbd>K</kbd>（或 <kbd>/</kbd>）召出索引，直达页面/篇章。
- **卷内检索**：命中高亮 + 上一处/下一处跳转；长卷不易迷路。
- **沉浸阅读**：界面“退后半步”，让内容站到台前。
- **拖拽排序**：藏品、收藏的次序可手动拉出；次序会被记住。
- **定位短路径**：洞府图/关系谱/纪事均可复制“定位：/path?...”写入札记或批注，一键回到现场。

> [!CAUTION]
> 若觉“卡卡的”，此站按默认策略持续降负：少重绘、少无谓动效、少重复计算——不加开关，不做臃肿模式。

<a id="vol-3"></a>

## 卷三 · 留印（落笔不散）

- 札记、收藏、批注、筛选状态等，默认落地保存（避免“刷新即丢”的幽灵）。
- 支持导出/导入：文卷（便于阅读与归档）/ 封卷（便于迁移与复原）。

> [!WARNING]
> 此卷偏向“个人经卷”：数据以浏览器本地为主；服务器只负责呈卷，不代管、不背负。

### 本地铸卷

安装依赖：

```bash
npm install
```

```bash
npm run dev
```

```bash
npm run build
```

```bash
npm run preview
```
