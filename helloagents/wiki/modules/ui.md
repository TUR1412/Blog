# ui

## 目的
设计系统的最小可复用基元：Button/Card/Chip/Badge 等，用统一的交互与视觉语言支撑全站。

## 模块概述
- **职责:** 统一按钮、卡片、标签等基础组件
- **状态:** ✅稳定
- **最后更新:** 2025-12-31

## 规范
### 需求: 统一交互手感
**模块:** ui
按钮与可交互组件需具备一致的 focus-ring、tap（触控手感）与 hover/active 反馈。

#### 场景: 键盘导航
在仅键盘操作时，焦点可见且不被背景吞没。
- 预期结果：focus-ring清晰、对比度足够

### 需求: 破坏性操作可识别
**模块:** ui
对“清空/覆盖/删除”等破坏性操作提供明确的视觉语义（danger），降低误触成本。

#### 场景: Confirm 对话框
当操作会覆盖或清空本地数据时，确认按钮应具备更强警示。
- 预期结果：danger 按钮与普通主按钮可一眼区分
